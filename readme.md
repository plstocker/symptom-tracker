# Symptom Tracker

This web app is a symptom tracker I built after being frusterated by existing apps that relied on ads or subscriptions to achieve something that could otherwise be accomplished with a pen and paper.

In building this app, I stuck to the following principles:

1. Logged data should be stored on the user's device.
2. Logging data should be as frictionless as possible.
3. The kinds of loggable data should be flexible.
4. The data should be accessible to the user.

### Logged Data Should be stored on the user's device.

All settings and logged data are stored in a databse that's stored locally on your phone. r That does mean that the permanance of the data is subject to whatever clean up rules exist on the device that is running it.

### Logging data should be as frictionless as possible.

This is the main reason I wanted to build this app. When you're experiencing a health event like a migraine or you're simply just trying to track potential triggers, its important to be able to do so quickly and without having to sort through files or papers. I designed this app to function like a remote. For most data entries you only need to touch a button and the app records the symptom, the date, the time, and the local weather. 

### The kinds of loggable data should be flexible.

Health needs change overtime, so I wanted to make sure that this web app could accomodate lots of differents needs and use-cases. Tracked symptoms fall into two categories: occurences and measurements. Measurements are further divided into single value and two value measurements, which accomodate reecording blood pressure or comparative measurments. Tabs and symptoms are fully customizable.


### The data should be accessible to the user.

Currently all data stored on the device by the app can be exported as a JSON or PDF. The app also allows for the import of JSON files, allowing the user to keep backups or switch devices.
